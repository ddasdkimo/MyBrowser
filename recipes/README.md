# Recipes

A **recipe** is a complete assembly path for a common vertical scenario. It pre-selects
the approach and components so the agent doesn't have to walk the full decision tree
from scratch.

A recipe is **not** a built artifact — it's the documented path to build one.

## Available recipes

- [`iseek-monitoring.md`](./iseek-monitoring.md) — a desktop browser tuned to run a
  multi-channel MJPEG CCTV monitoring dashboard perfectly, while still browsing the
  general web.

## Writing a new recipe

A recipe should state: the scenario, the chosen approach (+ why), the ordered list of
components to assemble, and which acceptance checklist verifies it.
